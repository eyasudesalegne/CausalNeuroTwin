# Governance

## Roles

- **Principal Investigator:** institutional accountability, scientific oversight, and approval of public scientific claims. The institutional PI is to be recorded before the first data-processing release.
- **Lead developer:** software architecture, implementation, testing, release preparation, and technical roadmap.
- **Maintainer:** issue triage, review, documentation, and release support.
- **Contributor:** accepted code, documentation, benchmark, review, or scientific contribution.

## Decisions

Routine implementation decisions are made through reviewed pull requests. Changes to scientific scope, licences, data governance, public model release, safety policy, or clinical interpretation require approval from the Principal Investigator and lead developer.

## Releases

A release requires:

- passing CI;
- documented changes;
- fixed environment and configuration information;
- reproducibility checks appropriate to the release;
- security and data-governance review;
- traceable release tag;
- archived artefact when appropriate.

## Authorship and credit

Git history, pull requests, release notes, and a contribution record will be used to document contributions. Publication authorship will follow actual scholarly contribution rather than repository access or job title alone.
